``pillar.get`` and friends (``pillar.items``, ``pillar.item``, ``pillar.raw``, ``pillar.ext``, ``pillar.ls``, ``pillar.obfuscate``) no longer mask string pillar values by default. The 3008 pillar-masking layer left these execution module entry points returning ``**********`` for plain string pillar values, forcing every caller — including states, returners, and operator scripts that hit pillar from a salt-call — to thread ``unmask=True`` through every call site. The execution module now returns plain values by default; operators who want masked output can opt in via the new ``pillar_mask_output: True`` minion config option. The render-time bracket that lets pillar SLS Jinja receive plain values via ``salt['pillar.get'](...)`` continues to work in both modes, and the ``MaskedDict``/``MaskedList`` repr safety net for incidental logging is unchanged.

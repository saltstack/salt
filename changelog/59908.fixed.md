Documented in ``CONTRIBUTING.rst`` that every new key added to
``DEFAULT_MASTER_OPTS`` / ``DEFAULT_MINION_OPTS`` /
``DEFAULT_PROXY_MINION_OPTS`` must also be documented in the
matching ``conf/`` sample and ``doc/ref/configuration/`` reference
page. Added ``tools/check_new_config_opts.py`` that flags drift
between the three sources, with unit tests at
``tests/pytests/unit/tools/test_check_new_config_opts.py`` and an
``--baseline`` flag so the hook only gates *new* drift, not the
historical undocumented backlog.

Allow a watching state to opt in to running ``mod_watch`` even when its own normal run already produced changes, by returning ``force_mod_watch: True``. The default behavior is preserved: if the normal run reports changes and does not opt in, ``mod_watch`` is still skipped, so non-idempotent watchers (such as ``cmd.run``) are not executed twice. When opted in, the ``changes`` and ``comment`` from ``mod_watch`` are merged into the normal run's result so neither set of changes is lost.

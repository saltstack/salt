Added a docstring note to ``salt.modules.haproxyconn`` flagging the
``TypeError: a bytes-like object is required, not 'str'`` failure
mode on Python 3 against upstream ``haproxyctl`` 0.3.1, pointing
at the upstream defect and listing the supported workarounds (pin
an older ``haproxyctl`` or drive the admin socket via
``cmd.run`` / ``socat``).

Set ``RELENV_DATA=/var/cache/salt/.relenv`` in the systemd units shipped with the RPM and Debian packages so relenv's bootstrap (run via ``relenv.pth`` on every Python startup) writes to a salt-owned location instead of the root-owned install dir. Previously, ``salt-master``, ``salt-minion``, ``salt-api``, and ``salt-syndic`` running as the non-root salt user produced a ``PermissionError: '/opt/saltstack/salt/.local'`` traceback on every CLI invocation.

Fixed ``pkgrepo.managed`` honouring ``clean_file: True`` when the desired
repo line is already present in the file. Previously the state returned
"already configured" and silently skipped both the file truncation and the
re-write, leaving any unrelated stale entries (e.g. an obsolete
``bullseye-backports`` line in a file managed for ``bookworm-backports``)
in place. The clean + reconfigure path now always runs when
``clean_file: True`` is requested.

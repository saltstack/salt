Fixed eauth tokens being evicted within seconds of issuance on 3008 regardless of `token_expire`. The generic `Cache.clean_expired()` fallback (used by the `localfs` driver) compared each entry's file mtime against the current time instead of its stored expiry, so the master's periodic token maintenance purged all still-valid tokens on every sweep -- breaking long-running salt-api operations such as batch jobs. `clean_expired()` now consults each entry's stored `_expires`, and `LoadAuth.mk_token` passes `token_expire` as a duration to `Cache.store` rather than an absolute timestamp.

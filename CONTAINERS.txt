in the sibling worktree:

  ../batching/CONTAINERS.txt

Open that path from this repo (same parent directory as this worktree). It describes
venv311/tools, container create/start, tools ts download test-artifacts, decompress-dependencies,

Onedir + nox inside Docker (bind mount must match host path)
-------------------------------------------------------------
After ``nox -e decompress-dependencies -- linux x86_64`` on the host, symlinks under
``.nox/ci-test-onedir/bin`` point at **absolute** paths like
``/home/you/src/salt/artifacts/salt/bin/python3``. If you mount the repo as ``-v $PWD:/salt``,
those symlinks break inside the container (``No such file or directory``). Mount the
checkout at the **same absolute path** on host and container, e.g.
``-v $REPO:$REPO -w $REPO``, then run
``$REPO/.nox/ci-test-onedir/bin/python -m pytest ...`` from inside the CI image.

Example (replace RUN_ID; slug ``ubuntu-24.04`` matches the Linux unit matrix):

  tools ts download test-artifacts RUN_ID ubuntu-24.04
  nox -e decompress-dependencies -- linux x86_64
  REPO=$(pwd)
  docker run --rm --privileged -v "$REPO:$REPO" -w "$REPO" \
    -e REPO="$REPO" -e SALT_TRANSPORT=zeromq -e ONEDIR_TESTRUN=1 \
    ghcr.io/saltstack/salt-ci-containers/testing:ubuntu-24.04 \
    /bin/bash -lc 'cd "$REPO" && .nox/ci-test-onedir/bin/python -m pytest \
      tests/pytests/unit/utils/test_nacl.py::test_enc_sk_file -vv --rootdir="$REPO"'

That reproduces the NaCl ``crypto_box_seal`` segfault (exit 139) seen in CI when
salt-factories log/event threads are active alongside PyNaCl.
